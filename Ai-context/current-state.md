**Landing rebuilt on real screenshots + a 10-bug truth sweep (Jul 12–13, `6f97090`→`d864522`)** — Started as "make the landing page work on mobile" and turned into an audit of every claim the product makes about itself. **All ten bugs below were live in production. Not one of them was in the Open Bugs table.**

- **The mobile homepage served the LOGIN FORM, not the landing page** — `showLoginAsHome` keyed off screen **WIDTH** (`max-width: 600px`) instead of launch context, so every phone visitor — and **Googlebot, which indexes mobile-first** — got a login form as the homepage. The entire top of funnel was strangled, silently, for months. Now keys off `display-mode: standalone` (+ iOS `navigator.standalone`): the installed PWA still opens on Login; a browser at any width gets the landing. `6f97090`.
- **Analytics followed visitors into the authenticated app** — `@vercel/analytics` `inject()` appends its script to `document.head` from an effect with **no cleanup**, so it outlives the `Landing` component; and `disableAutoTrack` is only set when a `route` prop is passed. Mounted bare, it kept auto-tracking history changes — reporting `/log`, `/profile` and `/client/<uuid>` pageviews. `Privacy.jsx` claimed *"this analytics does not run on signed-in or health-data routes"* — which was **false**, and **a false statement in a privacy policy about health data is the FTC violation itself** (cf. GoodRx, BetterHelp). Two guards in `Landing.jsx`: `route`/`path` props (which set `data-disable-auto-track` on the script tag) and a **fail-closed `beforeSend` path allowlist**. `6ba8217`, `87f5041`.
- **The page advertised a paywall that is switched off** — `BILLING_ENABLED = false`, so `CoachPaywall` (the only coach path to `create-checkout-session`) never renders: coaches sign up free, no card, no charge. The page said "$19/month", "Card required to start", "not charged until day 31". **The copy was inferred from `create-checkout-session` without ever checking whether the paywall that calls it was reachable.** It wasn't. Copy is now what's true: free while in early access, no card, **no price named at all**. `9a98ac8`, `1697fec`.
- **Stripe took its price from the BROWSER** — `create-checkout-session` read `priceId` off the request body and put it straight into `line_items[0][price]`. The **role** was verified against the DB; the **price** was verified against nothing — and both price IDs shipped in the JS bundle (`VITE_*` is public), so a coach could have subscribed at the **solo** rate. Now resolved server-side from the verified role via a `PRICE_IDS` map over `Deno.env`, failing closed if unconfigured; the client sends no price at all. `8079e1f` (v33).
- **Solo AI feedback 403'd EVERY free solo user — the entire free funnel** — `SOLO_BILLING_ENABLED = false` makes `hasSoloPremium` always true, so the client offered "Get AI feedback" to every solo user; `nutrition-coach` never got the memo and still demanded a paid subscription, returning 403 *"Solo Premium required"*. **The two halves of the flag disagreed about whether Solo is free.** The check now sits behind a `SOLO_BILLING_ENABLED` env var **on the function, defaulting to FREE**; what guards that endpoint is the 30/hr rate cap + response cache, not a paywall. Verified **both directions** on a local stack — flag off, a free solo user with zero subscription rows clears the gate; flag on, the 403 returns. `51b9060` (v23).
- **The coach dashboard's "Need attention" tile lied** — it counts RED only (deliberate — `attentionLevel.js` pins its threshold to it) but sat directly beneath a banner counting red **and** amber.
